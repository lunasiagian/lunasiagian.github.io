SAKAPANA KNOWLEDGE GRAPH

PROBLEM STATEMENT

A knowledge graph of Sakapana cannot be a static graph; edges appear and disappear as fog events close and reopen passages. Traditional static modeling fails to capture the physical reality of the Collegium's changing infrastructure.

WHAT WAS BUILT

A temporal state system using NetworkX for underlying logic and D3.js for visual rendering. The system treats nodes (rooms, terminals, passages) as constant, but evaluates the probability of edge connections dynamically based on a cryptographic time-hash of the current epoch.

WHAT IT DOES / DOESN'T DO

DOES: Successfully models temporal access. If the fog state reaches "major", the graph isolates the outer wards automatically.
DOESN'T: The system does not yet support full text search on node metadata, limiting its use as a primary archival tool.

WHAT WAS LEARNED

When you build a system where the data structure physically alters itself based on environmental conditions, you stop treating it like a database and start treating it like an organism. The logic required to manage "disappearing data" without triggering system fatal errors taught me that graceful degradation isn't just a UI principle—it's a fundamental architectural requirement for resilient systems.
